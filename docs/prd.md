# 需求文档

## 1. 应用概述

### 1.1 应用名称
影刀社区数据看板（动态版）

### 1.2 应用描述
将现有的影刀社区数据看板静态HTML网页升级为公网可访问的动态数据看板网站，支持通过影刀RPA流程自动采集并写入数据，前端实时从数据库读取并展示数据，保留原有深色科技风格和所有图表内容。

## 2. 用户与使用场景

### 2.1 目标用户
- 影刀社区运营人员：查看社区问答数据趋势和用户反馈
- 影刀产品团队：分析用户问题分布，制定产品优化方向
- 影刀RPA流程开发者：通过API接口自动提交采集的帖子数据

### 2.2 核心使用场景
- 影刀RPA流程定期采集社区帖子数据，通过HTTP接口写入数据库
- 用户访问看板页面，查看实时数据统计和图表分析
- 管理员通过数据管理页面查询、筛选、手动添加帖子数据

## 3. 页面结构与功能说明

### 3.1 页面结构

```
影刀社区数据看板网站
├── 数据看板页面（/）
├── 数据管理页面（/admin）
└── 数据接收接口（后端API）
```

### 3.2 数据看板页面（/）

#### 3.2.1 页面头部
- 显示标题：影刀社区数据看板
- 显示数据来源说明：影刀社区问答界面帖子,通过影刀RPA流程采集
- 右上角显示最后更新时间和数据刷新状态
- 提供「立即刷新」按钮,点击重新拉取最新数据

#### 3.2.2 筛选区域
- 日期范围筛选：默认显示最近7天,支持自定义选择日期范围
- 标签分类筛选：支持选择T1网页交互/T2指令报错/T3数据处理/T4第三方对接/T5功能咨询,支持多选

#### 3.2.3 KPI指标卡片区域
显示以下实时计算的指标卡片：
- 总帖子数
- 浏览量最高帖子的浏览数
- 各标签分类的帖子数量（T1/T2/T3/T4/T5）
- 情感倾向分布（积极/中性/消极的百分比）

#### 3.2.4 图表区域
- 每日活跃度趋势图：折线图或柱状图,展示每日发帖数量趋势
- 问题类型分布图：柱状图或环形图,展示各标签分类的帖子数量占比
- 情感倾向分析图：环形图,展示积极/中性/消极情感的占比
- 问题类型详细拆解表：表格形式,展示各标签分类的详细数据

#### 3.2.5 热门帖子TOP10表格
- 展示浏览量最高的前10条帖子
- 显示字段：帖子标题、帖子作者、浏览量、标签分类、发帖日期
- 支持分页查看更多帖子
- 点击帖子标题可查看帖子详情（标题、内容、作者、浏览量、标签、情感倾向、发帖日期）

#### 3.2.6 核心洞察卡片
自动从数据计算生成以下洞察：
- 帖子数量最多的标签分类
- 发帖数量最多的星期几
- 情感倾向占比最高的类型

#### 3.2.7 行动建议清单
展示6条可落地行动方案（内容从现有HTML中保留）

#### 3.2.8 页面底部
- 显示最后更新时间
- 显示数据总量

### 3.3 数据管理页面（/admin）

#### 3.3.1 帖子列表
- 展示数据库中所有帖子的列表
- 显示字段：id、帖子标题、帖子作者、浏览量、标签分类、情感倾向、发帖日期

#### 3.3.2 筛选查询
- 支持按日期范围筛选
- 支持按标签分类筛选
- 支持按情感倾向筛选

#### 3.3.3 手动添加帖子
- 提供表单,支持手动输入单条帖子数据
- 表单字段：帖子标题、帖子内容、帖子作者、浏览量、标签分类（下拉选择T1/T2/T3/T4/T5）、情感倾向（下拉选择positive/neutral/negative）、发帖日期
- 提交后写入数据库

#### 3.3.4 接口文档说明
- 显示影刀RPA接口的调用地址：/functions/v1/receive-posts
- 显示接口请求方式：POST
- 显示接口认证方式：请求头中传入x-api-key
- 显示接口请求格式示例（JSON格式的帖子数组）
- 显示接口返回格式示例（成功数量、失败原因）

### 3.4 数据接收接口（后端API）

#### 3.4.1 接口路径
/functions/v1/receive-posts

#### 3.4.2 接口功能
- 接收影刀RPA流程提交的帖子数据
- 支持批量写入（一次提交多条帖子）
- 验证请求头中的x-api-key
- 将数据写入posts表
- 返回写入结果（成功数量、失败原因）

#### 3.4.3 请求格式
- 请求方式：POST
- 请求头：x-api-key（API密钥）
- 请求体：JSON格式的帖子数组,每条帖子包含title、content、author、views、tag、sentiment、post_date字段

#### 3.4.4 返回格式
- 成功：返回写入成功的数量
- 失败：返回失败原因

## 4. 数据库设计

### 4.1 posts表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 自增主键 | 帖子唯一标识 |
| title | 文本 | 帖子标题 |
| content | 文本 | 帖子内容 |
| author | 文本 | 帖子作者 |
| views | 整数 | 浏览量 |
| tag | 文本 | 标签分类（T1/T2/T3/T4/T5） |
| sentiment | 文本 | 情感倾向（positive/neutral/negative） |
| post_date | 日期 | 发帖日期 |
| created_at | 时间戳 | 数据创建时间 |

## 5. 业务规则与逻辑

### 5.1 数据流转逻辑
- 影刀RPA流程采集社区帖子数据 → 通过HTTP接口提交数据 → 数据接收接口验证并写入数据库 → 前端从数据库读取数据并展示

### 5.2 数据筛选逻辑
- 日期范围筛选：根据post_date字段筛选指定日期范围内的帖子
- 标签分类筛选：根据tag字段筛选指定标签的帖子,支持多选
- 情感倾向筛选：根据sentiment字段筛选指定情感倾向的帖子

### 5.3 KPI指标计算逻辑
- 总帖子数：统计筛选条件下的帖子总数
- 浏览量最高帖子浏览数：筛选条件下views字段的最大值
- 各标签分类帖子数量：按tag字段分组统计
- 情感倾向分布：按sentiment字段分组统计,计算百分比

### 5.4 核心洞察计算逻辑
- 帖子数量最多的标签分类：按tag字段分组统计,取数量最多的分类
- 发帖数量最多的星期几：根据post_date字段提取星期几,按星期分组统计,取数量最多的星期
- 情感倾向占比最高的类型：按sentiment字段分组统计,取占比最高的类型

### 5.5 数据刷新逻辑
- 前端每5分钟自动重新拉取数据
- 用户点击「立即刷新」按钮时,立即重新拉取数据
- 更新最后更新时间显示

### 5.6 接口认证逻辑
- 数据接收接口验证请求头中的x-api-key
- 若x-api-key不正确,返回认证失败错误

- 表格样式
- 核心洞察卡片样式

## 7. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 数据库无数据 | 显示「暂无数据」提示 |
| 筛选条件下无数据 | 显示「暂无符合条件的数据」提示 |
| 接口请求失败 | 显示错误提示,提供重试按钮 |
| x-api-key验证失败 | 返回认证失败错误 |
| 批量写入部分失败 | 返回成功数量和失败原因 |
| 手动添加帖子时必填字段为空 | 显示字段验证错误提示 |
| 日期范围选择不合法 | 显示日期格式错误提示 |

## 8. 验收标准

1. 影刀RPA流程通过HTTP接口成功提交帖子数据,数据写入posts表
2. 访问数据看板页面,页面展示实时数据,包含KPI卡片、图表、热门帖子TOP10表格、核心洞察卡片、行动建议清单
3. 在数据看板页面选择日期范围和标签分类筛选条件,页面数据根据筛选条件动态更新
4. 点击「立即刷新」按钮,页面数据重新拉取并更新,最后更新时间显示为当前时间
5. 访问数据管理页面,查看帖子列表,使用筛选条件查询帖子,手动添加一条帖子数据,数据成功写入数据库
6. 在数据管理页面查看接口文档说明,复制接口地址和请求示例,配置影刀HTTP节点成功调用接口
